Mere fmt — written in Mere

This page runs Mere code that pretty-prints Mere AST. The Wasm module below was built from contrib/fmt/fmt.mere (~510 lines of Mere) — a port of the OCaml Mere.Formatter covering precedence-driven paren insertion, block-form layout, else if chain flattening, and sugar reconstruction (range a b → a..b, Cons/Nil chain → list literal, fn x -> fn y -> body → \x y -> body).

Click Run to instantiate the Wasm module and execute its built-in demo: it formats six hand-coded ASTs and prints the results. The output is what the self-host fmt produces from each AST literal.

Cross-validation

For the factorial AST below, the self-host fmt produces byte-identical output to mere fmt on the equivalent source. Run the demo to verify the right-hand side matches the left.

mere fmt (OCaml side, run separately): 
$ echo 'let rec fact = fn n -> if n < 1 then 1 else n * fact (n - 1) in fact 10' \
  | mere fmt /dev/stdin

let rec fact = fn n -> if n < 1 then 1 else n * fact (n - 1) in
fact 10
Mere self-host fmt (this page's "factorial" line): 
let rec fact = fn n -> if n < 1 then 1 else n * fact (n - 1) in
fact 10

Why this matters

Self-hosting the formatter is the first concrete step of Mere's Phase 49 / §S1 plan (contrib/fmt/README.md):

  • Stage 49a — Mere AST expressed as a Mere variant tree
  • Stage 49b — full pretty-printer ported from OCaml
  • Stage 49d — this page (Wasm + browser demo)
  • Stage 49c — Mere parser written in Mere (Phase 50; would let this demo accept arbitrary source instead of hand-coded AST literals)

The parser is the real self-host bottleneck. Once it lands, this page becomes a real in-browser mere fmt — type source, see formatted output, no server round-trip.
