# contrib/site — Mere docs site SSG (Static Site Generator) MVP

A docs site builder written in Mere itself. Generates HTML pages + index from
a markdown directory. The flagship dogfood lib that backs "Mere docs are
generated by Mere itself" at public release.

## Files

| file | content | lines |
|---|---|---|
| `build.mere` | SSG core (CLI script taking input/output dir as args) | ~145 |

## Usage

```sh
# one-shot build (markdown only; playground/*.wasm needs separate cp)
dune exec mere -- contrib/site/build.mere docs _site

# full build (including Wasm playground) — via shell wrapper
sh contrib/site/build_full.sh docs _site

# dev mode: inject <meta refresh content="2"> into HTML (browser reloads every 2s)
sh contrib/site/build_full.sh docs _site --dev

# watch mode: dev mode + 500ms polling detects mtime change in docs/ and auto-rebuilds
sh contrib/site/build_full.sh docs _site --watch
# In another terminal: `python -m http.server -d _site 8000` → http://localhost:8000/
```

When args are omitted, default is `docs/` → `_site/`. `build_full.sh` is a
wrapper combining the Mere SSG with binary `.wasm` copies (because Mere's
`read_file` / `write_file` are UTF-8 string-based and can't binary-copy).

## MVP scope

| feature | status |
|---|---|
| enumerate `*.md` in input dir | ✓ (`list_dir` + `Path.has_ext`) |
| markdown → HTML conversion | ✓ (`MarkdownHtml.render`) |
| page title extraction (leading `# Title` line, or filename) | ✓ |
| common template (header / nav / footer) | ✓ (external `style.css` + `<link>` reference) |
| navigation menu (all pages in a row, current page highlighted via `active` class) | ✓ |
| `index.html` (page list) | ✓ |
| auto-create output dir | ✓ (`mkdir_p`) |
| auto-generated external `style.css` (60+ lines: nav / table / code / blockquote / list / search styles) | ✓ |
| fenced code block language class (`<code class="language-mere">` etc.) | ✓ (`mere`/`sh`/`ml` etc.) |
| client-side search (`search.json` + vanilla JS full-text search with excerpt display) | ✓ |
| front matter (reduced YAML: `--- title: ... date: ... ---`) | ✓ (title override supported) |
| per-page TOC (auto-generated from h2/h3 with anchor links, collapsible `<details>`) | ✓ (non-ASCII alphanumerics slugified to `-`; identical slugs may collide for Japanese etc.) |
| auto-generated `sitemap.xml` | ✓ (relative URLs, for SEO / crawlers) |
| auto-placed `.nojekyll` (disables GitHub Pages Jekyll processing) | ✓ |
| **Wasm playground** (pre-compiled Mere → Wasm + browser-side loader) | ✓ (`playground/*.html` + `*.wat` copied by SSG; `*.wasm` copied via `build_full.sh`) |
| **syntax highlighting** (Prism.js + custom `prism-mere.js`) | ✓ (CDN core + clike + bash + ocaml + custom Mere definition, CSS override to match docs palette) |

## Beyond MVP (future Phases)

- multi-version / i18n (per-language docs, versioned URLs)
- live reload — websocket-based push reload (currently polling watch mode)
- static copy of images (currently only `style.css` is auto-emitted; needed when docs add images)
- lunr.js-based search index (currently substring filter; needed when docs grow large)
- math rendering via katex / mathjax
- diagram rendering via mermaid etc.

## Backend support

| backend | status |
|---|---|
| interp | ✓ |
| C | ✓ (Phase 44 added list_dir / mkdir_p to C codegen) |
| LLVM | ✗ (list_dir / mkdir_p not implemented for LLVM) |
| Wasm | ✗ (list_dir / mkdir_p not implemented for Wasm) |

The MVP mainly runs **through interp**. For native binary, use C codegen.

## Known limitations

- **The demo code in imported libs (`MarkdownHtml` / `Path`) also runs** at
  build time (each lib file has self-tests at top level). Extra prints leak
  into build output but don't affect SSG output.
- MarkdownHtml doesn't support fenced code blocks (``` ``` ```); requires
  extending the markdown lib itself in the future.
